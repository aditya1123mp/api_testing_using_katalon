# API Testing with Katalon Studio

This repository contains automated test scripts for verifying API functionality using Katalon Studio. The tests cover a range of scenarios to ensure the API is functioning as expected.

## Test Cases

The following test cases are automated in this project:

1. **Verify Status Code**  
   Ensure the API response status code is `200 OK`.

2. **Verify Response Format**  
   Validate that the API response is in the expected format (e.g., JSON, XML).

3. **Verify Expected Fields**  
   Check that the API response contains all the expected fields.

4. **Verify Field Data**  
   Confirm that the API response contains the correct data for each field.

5. **Verify Response Time**  
   Ensure the API response time is within acceptable limits.

6. **Verify Request Parameters**  
   Validate that the API request parameters are correctly passed to the API.

7. **Verify Request Method**  
   Check that the API request method is correct (e.g., GET, POST, PUT, DELETE).

8. **Verify Endpoint URL**  
   Ensure the API endpoint URL is correct.

9. **Verify Response Headers**  
   Validate that the API response headers are correct.

10. **Verify Payload Size**  
    Confirm that the API response payload size is within acceptable limits.

11. **Verify Error on Malformed Request**  
    Check that the API returns an error message if the request is malformed.

12. **Verify Error on Authentication Failure**  
    Ensure the API returns an error message if authentication fails.

13. **Verify Error on Missing Payload**  
    Validate that the API returns an error message if the request payload is missing.

14. **Verify Error on Non-Existent Resource**  
    Confirm that the API returns an error message if the requested resource does not exist.

15. **Verify Error on Unauthorized Resource Access**  
    Ensure the API returns an error message if the requested resource is not authorized.

16. **Verify Error on Exceeding Payload Limit**  
    Validate that the API returns an error message if the request payload exceeds the allowed limit.

17. **Verify Error on Invalid Data**  
    Confirm that the API returns an error message if the request payload contains invalid data.

18. **Verify Error on Incorrect Method**  
    Check that the API returns an error message if the request method is not allowed for the resource.

19. **Verify Success on Resource Creation**  
    Ensure the API returns a success message if the resource is created successfully.

20. **Verify Success on Resource Update**  
    Validate that the API returns a success message if the resource is updated successfully.

21. **Verify Success on Resource Deletion**  
    Confirm that the API returns a success message if the resource is deleted successfully.

22. **Verify Success on Resource Retrieval**  
    Ensure the API returns a success message if the resource is retrieved successfully.

23. **Verify Correct Resource by Identifier**  
    Check that the API returns the correct resource based on the provided resource identifier.

24. **Verify Correct Resource by Search Parameters**  
    Validate that the API returns the correct resource based on the provided search parameters.

25. **Verify Pagination Information**  
    Ensure the API response contains the correct pagination information.
